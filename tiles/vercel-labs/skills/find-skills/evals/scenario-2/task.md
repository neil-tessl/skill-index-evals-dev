# Finding Skills for Embedded Firmware Development

## Problem/Feature Description

A hardware engineer has just started using an AI coding assistant and has heard that it can be extended with installable skills. They work in embedded firmware — writing low-level C/C++ for microcontrollers, dealing with JTAG debugging, linker scripts, and real-time operating systems like FreeRTOS. It's a niche domain and they're not sure whether the skills ecosystem has anything useful for them.

They've asked you to do a proper search and come back with whatever you can find. If nothing relevant turns up, they want to know that too, along with any advice on how they might still get help with their work.

## Output Specification

Produce a file called `firmware-skills-search.md` that documents:

1. The search strategy you used (queries attempted)
2. The outcome of each search
3. Your conclusion — whether relevant skills were found and your recommendation
4. If no suitable skills were found: what the developer can do instead, and how they could address the gap longer term
